# RogueLoopKit Cursor Rules

## Project Goal
This project is a modular mobile roguelike starter kit for Unity 6 URP 2D.

The architecture must prioritize:
- readability
- modularity
- scalability
- inspector friendliness
- mobile performance
- fast iteration

This is NOT an enterprise framework.
Avoid overengineering.

---

# Coding Rules

## General
- Use clean and simple C#
- Prefer composition over inheritance
- Avoid unnecessary abstractions
- Avoid premature optimization
- Avoid complex patterns unless necessary

---

## Architecture
- Keep scripts small and focused
- One responsibility per component
- Avoid god classes
- Avoid giant managers
- Prefer ScriptableObjects for configuration
- Prefer event-driven UI

---

## Unity Rules
- Use SerializeField instead of public fields
- Avoid FindObjectOfType in Update
- Cache references
- Use prefabs correctly
- Keep scene hierarchy organized

---

## Mobile Rules
- Design for mobile first
- Avoid unnecessary allocations
- Avoid expensive Update loops
- Keep gameplay readable on small screens

---

## Code Style
- Use clear naming
- Use headers for inspector organization
- Add short comments only when necessary
- Avoid magic numbers
- Keep methods short

---

## AI Assistant Behavior
When generating code:
- explain architecture decisions briefly
- prefer maintainability over cleverness
- avoid unnecessary systems
- avoid enterprise patterns
- avoid dependency injection frameworks
- avoid ECS unless explicitly requested

Always generate production-usable Unity code.
